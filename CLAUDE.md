# CLAUDE.md - プロジェクトルールテンプレート

## プロジェクト概要
- **プロジェクト名**: 温湿度モニタリングアプリ
- **目的**: DHTセンサーからのデータ取得方法の習得、PythonによるI/O制御とデータ処理技術の学習、データの記録・可視化準備としてCSV出力手法の理解
- **対象**: プログラミング初心者、Raspberry Pi入門者、センサー制御を学びたい電子工作学習者

## 開発ルール

### 業務開始時の必須確認事項
- **要件定義書の確認**: 業務開始時は必ず要件定義書を読み返し、最新の要件と仕様を確認する
- **プロジェクト状況把握**: 現在の進捗状況と残タスクを確認する
- **変更履歴確認**: 前回からの変更点や更新事項をチェックする

### コーディング規約
- **言語**: Python 3.9以上
- **コーディングスタイル**: PEP8準拠
- **ライブラリ**: Adafruit_DHT, argparse（標準）, csv（標準）, time（標準）
- **コーディングガイドライン準拠**:
  - `python_coding_guidelines.md`に記載されたコーディング規約に従う
  - `if __name__ == "__main__":`を必ず使用
  - 関数にdocstringを記載
  - try-finallyでクリーンアップを確実に実行
  - 具体的な例外処理を実装
  - インポートの順序（標準→サードパーティ→自作）を守る
- **コメント規約**:
  - `COMMENT_STYLE_GUIDE.md`に記載されたコメント規約に従う
  - 日本語でわかりやすくコメントを記入する
  - ファイルヘッダーに要件定義書への参照を記載
  - クラス・メソッドに適切なdocstringを記載
  - 初心者向けにライブラリの仕様や技術的背景を説明
  - エラーメッセージに対処方法と具体的なコマンド例を記載
  - 複雑な処理や重要な判定部分には必ずコメントを追加
  - トラブルシューティング情報を含める
- **関数分割**: 機能ごとに適切に分割（setup、main、cleanup）
- **エラーハンドリング**: 必須実装、初心者にわかりやすいメッセージ

### セキュリティルール
- **権限**: 最小権限の原則
- **入力検証**: 全ての引数とパラメータを検証
- **エラー情報**: 機密情報の漏洩防止
- **機密情報管理**: 
  - ユーザー名、パスワード、APIキー、秘密鍵などの機密情報をGitHubにアップロードしない
  - 設定ファイルや環境変数ファイル（.env等）は.gitignoreに追加
  - コード内にハードコードされた機密情報がないことを確認
- **リポジトリ更新前チェック**: 
  - コミット前に必ず機密情報が含まれていないかコードレビューを実施
  - git add前にファイル内容を再確認
  - .gitignoreの設定が適切であることを確認

### テストルール
- **機能テスト**: 実装した機能の動作確認必須
- **引数テスト**: 各コマンドライン引数の検証
- **エラーテスト**: 異常系の動作確認
- **割り込みテスト**: 適切な終了処理の確認

### ドキュメントルール
- **README**: 英語で記載し、安全注意事項を必ず記載
- **使用例**: 具体的なコマンド例を提供
- **トラブルシューティング**: よくある問題と解決方法

### 運用ルール
- **バックアップ**: 重要な変更前にバックアップ
- **バージョン管理**: 機能追加時はバージョン番号更新
- **ログ**: タイムスタンプ付きで動作状況記録
- **終了処理**: 必ずリソースのクリーンアップ実行

## 禁止事項
- 機密情報のハードコーディング
- 適切な権限なしでのリソース操作
- クリーンアップ処理の省略
- GPIO操作時の不適切な電圧印加
- センサー接続時の回路ショート
- 長時間実行時のリソースリーク

## 推奨事項
- 定期的なコードレビュー
- 実装機能のテスト確認
- エラーログの適切な管理
- 教育効果を考慮した実装
